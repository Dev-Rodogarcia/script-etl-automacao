### 4. Melhoria na Proteção de Paginação

**Problema Identificado:**
- Proteção de cursor repetido estava interrompendo prematuramente
- Sistema tratava última página válida como loop infinito
- Extração de `usuarios_sistema` interrompida com apenas 40 registros

**Solução Implementada:**

**Lógica Aprimorada em `ClienteApiGraphQL.java`:**
- Verifica se página retornou menos registros que o esperado
- Se página incompleta + cursor repetido → trata como última página válida
- Se página completa + cursor repetido + hasNextPage=true → loop infinito (interrompe)

**Código Implementado:**
```java
if (novoCursor != null && cursor != null && novoCursor.equals(cursor)) {
    if (resposta.getHasNextPage()) {
        final int registrosEsperados = perInt; // first: 100
        final int registrosRecebidos = resposta.getEntidades().size();
        
        if (registrosRecebidos < registrosEsperados) {
            // Página incompleta + cursor repetido = última página válida
            logger.warn("⚠️ Entidade {}: Cursor repetido mas página incompleta. Tratando como última página.", 
                nomeEntidade);
            break; // Não interromper
        } else {
            // Loop infinito detectado
            interrompido = true;
            break;
        }
    }
}
```

### 5. Atualização da View PowerBI para Coletas

**Problema Identificado:**
- View `vw_coletas_powerbi` fazia JOIN apenas com `destroy_user_id`
- Campo `cancellation_user_id` não tinha enriquecimento com nome do usuário
- Campos aparecendo como NULL mesmo quando dados existiam

**Solução Implementada:**

**Arquivo:** `database/views/013_criar_view_coletas_powerbi.sql`

**Mudanças:**
- ✅ Adicionado `LEFT JOIN` com `dim_usuarios` para `cancellation_user_id` (alias `u_cancel`)
- ✅ Adicionado campo `[Usuario Cancel. Nome]` com `COALESCE`
- ✅ Mantido `LEFT JOIN` para `destroy_user_id` (alias `u_destroy`)
- ✅ Campo `[Usuario Exclusao Nome]` já existia, mantido

**Código Atualizado:**
```sql
FROM dbo.coletas c
LEFT JOIN dbo.dim_usuarios u_cancel ON c.cancellation_user_id = u_cancel.user_id
LEFT JOIN dbo.dim_usuarios u_destroy ON c.destroy_user_id = u_destroy.user_id;
```
