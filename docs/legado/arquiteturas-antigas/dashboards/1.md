Aqui está um **Prompt "Master"** extremamente detalhado e técnico.

Você pode copiar e colar isso para mim (ou para qualquer IA de codificação) para garantir que o código gerado já venha com uma arquitetura profissional, visualmente rica e desacoplada, usando **Plotly Dash + Bootstrap**, fugindo do visual "padrão" do Streamlit.

-----

### 📋 Copie e Cole o Prompt Abaixo:

```markdown
Atue como um Engenheiro de Software Sênior especializado em Python e Data Visualization.

**Objetivo:**
Criar a estrutura base de um Dashboard Financeiro Corporativo ("Enterprise Level") utilizando **Python**, substituindo uma solução anterior em Streamlit. O objetivo é ter total liberdade de layout, design responsivo e uma aparência profissional (estilo SaaS bancário).

**Stack Tecnológica Obrigatória:**
1.  **Framework:** Plotly Dash (não usar Streamlit).
2.  **UI/Design:** Dash Bootstrap Components (DBC) para grid system (Rows/Cols) e componentes estilizados (Cards, Modais, Botões).
3.  **Gráficos:** Plotly Express / Plotly Graph Objects (com temas limpos/white).
4.  **Dados:** Pandas + SQLAlchemy (para conexão com SQL Server).
5.  **Estilização:** Uso de folha de estilos externa (tema BOOTSTRAP ou LUX) + CSS customizado se necessário.

**Estrutura de Dados (Contexto):**
O dashboard deve consumir dados de duas tabelas SQL Server já validadas:
1.  `faturas_a_pagar_data_export` (Contas a Pagar).
2.  `faturas_por_cliente_data_export` (Contas a Receber/Fretes).

**Requisitos de Layout e UX (O que eu quero diferente do Streamlit):**
1.  **Sidebar Fixa:** Menu lateral esquerdo escuro com navegação (Visão Geral, Contas a Pagar, Contas a Receber).
2.  **Cards de KPI no Topo:** 4 cartões lado a lado (Grid 1x4) mostrando: Total a Pagar, Total a Receber, Saldo Projetado, Títulos em Atraso. Use ícones se possível.
3.  **Layout Livre:** Quero uma linha com 2 gráficos (60% largura / 40% largura) e abaixo uma tabela detalhada ocupando 100%.
4.  **Filtros Globais:** Um painel superior com DatePickerRange (Seleção de datas) e Dropdown de Filial que afete todos os gráficos via Callbacks.

**Entregáveis Esperados:**
1.  **Estrutura de Pastas:** Sugira uma organização de arquivos (ex: `app.py`, `/components`, `/data`, `/assets`) para não ficar tudo em um arquivo só.
2.  **Código do `app.py`:** O ponto de entrada.
3.  **Código do Layout (`layout.py`):** Onde a estrutura visual é montada usando `dbc.Container`, `dbc.Row`, `dbc.Col`.
4.  **Código dos Callbacks (`callbacks.py`):** A lógica que atualiza os gráficos.
5.  **Conexão de Dados:** Uma função de exemplo robusta usando `pd.read_sql` com cacheamento simples.

Por favor, capriche no visual. Quero que pareça um sistema profissional, com espaçamentos (padding/margin) corretos e sombras suaves nos cards (shadow-sm).
```

-----

### 💡 Por que este prompt funciona?

1.  **Define a "Stack de Beleza":** Ao exigir `Dash Bootstrap Components`, você garante que o layout não será aquela "linguiça" vertical do Streamlit. Ele usará o sistema de grid de 12 colunas (igual sites profissionais).
2.  **Exige Arquitetura:** Ao pedir a "Estrutura de Pastas", você evita receber um script de 500 linhas que é impossível de dar manutenção depois.
3.  **Contexto de Negócio:** Você já passa os nomes das tabelas (`faturas_a_pagar...`), então o código gerado já virá com as queries SQL prontas para o seu banco.
